# MaintainHome тАФ Native iOS / Android (Capacitor)

The web app at `artifacts/maintain-home` is now wrapped with [Capacitor 8](https://capacitorjs.com)
so it can ship to the App Store and Google Play as a native iOS / Android app.

## What's already in place

- **Capacitor core + CLI** (`@capacitor/core`, `@capacitor/cli`)
- **Native platforms**: `ios/` and `android/` folders generated by `cap add`
- **Plugins installed**:
  - `@capacitor/camera` тАФ camera + photo library
  - `@capacitor/filesystem` тАФ document upload / local file access
  - `@capacitor/push-notifications` тАФ APNs / FCM push
  - `@capacitor/splash-screen` тАФ branded launch image
  - `@capacitor/app` тАФ app lifecycle events
  - `@capacitor/preferences` тАФ secure key/value storage
  - `@capacitor/status-bar` тАФ colour/style of the status bar
- **Permissions wired up** in `ios/App/App/Info.plist` and
  `android/app/src/main/AndroidManifest.xml` for camera, photos, microphone,
  storage, and push notifications.
- **Placeholder app icon + splash** generated from `public/images/logo.png`.
- **Bundle ID**: `com.maintainhome.app`
- **App name**: `MaintainHome`
- **Web bundle dir**: `dist/public`

See `capacitor.config.ts` for the full config.

## What you need to do locally

Capacitor needs **Xcode** (Mac only) and **Android Studio** to actually build,
run, and submit. Replit's Linux container can't open these projects, so the
day-to-day native dev loop is on your laptop.

### One-time setup on your machine

1. Pull the repo to your Mac.
2. Install dependencies:
   ```sh
   pnpm install
   ```
3. **iOS only** тАФ install Xcode (App Store) and CocoaPods:
   ```sh
   sudo gem install cocoapods
   cd artifacts/maintain-home/ios/App
   pod install
   cd -
   ```
4. **Android only** тАФ install [Android Studio](https://developer.android.com/studio)
   and let it install the Android SDK + an emulator.

### Day-to-day workflow

From inside `artifacts/maintain-home`:

```sh
# 1. Build the web bundle and copy it into the native projects.
#    (PORT + BASE_PATH are required because vite.config.ts reads them.)
pnpm cap:sync

# 2. Open Xcode (iOS) тАФ click тЦ╢ to run on simulator/device.
pnpm cap:open:ios

# 3. Open Android Studio тАФ click the green тЦ╢ button.
pnpm cap:open:android
```

`pnpm cap:sync` runs `vite build` (writes to `dist/public/`, the
`webDir` declared in `capacitor.config.ts`) and then `cap sync`, which
copies the web assets and updated plugin native code into both the
`ios/` and `android/` folders.

### Verifying the production API rewrite

After `pnpm cap:sync`:

1. **iOS simulator (Xcode)**
   - Run the app, then in Safari open
     **Develop тЖТ Simulator тЖТ MaintainHome** to attach the Web Inspector.
   - In the Console you should see:
     `[Capacitor] Running in native mode тАФ using production API: https://maintainhome.ai`
   - In the Network tab confirm requests go to `maintainhome.ai`, not
     `capacitor://` or `localhost`.

2. **Android emulator (Android Studio)**
   - Run the app, then open `chrome://inspect` in Chrome on your Mac
     and "Inspect" the WebView.
   - Same console + network checks as above.

3. **Browser sanity check**
   - `pnpm dev` and confirm the Console does **not** show the
     "Running in native mode" log and that requests stay relative
     (`/api/...`). The patch is gated by `Capacitor.isNativePlatform()`,
     so the PWA path is unaffected.

### Refreshing icons or splash screens

The placeholder icons are generated from `public/images/logo.png`. To swap
them out, drop new source files into `assets/` (`icon-only.png`,
`icon-foreground.png`, `icon-background.png`, `splash.png`,
`splash-dark.png` тАФ all square, 1024px+ for icons, 2732px for splash) then
run:

```sh
pnpm cap:assets
pnpm cap:sync
```

(`pnpm cap:assets` requires `sharp`'s native binary; it works on a Mac out
of the box, but on Replit you'd need to use the ImageMagick fallback that
was used for this initial setup.)

## API URL handling in native builds тАФ done

When the app runs natively, `fetch("/api/...")` would resolve to
`capacitor://` (iOS) or `http://localhost` (Android) instead of the real
backend. To avoid touching every component, `src/lib/api.ts` exports a
small `installNativeFetchRewrite()` helper that **monkey-patches
`window.fetch` at app start** and rewrites any relative or
capacitor/localhost URL to `https://maintainhome.ai`. It is a no-op in the
browser/PWA build.

The patch runs once from `src/main.tsx` before React mounts, so every API
call (raw `fetch`, React Query, Axios using fetch adapter, etc.) is
covered automatically. On startup in a native build you'll see this log:

```
[Capacitor] Running in native mode тАФ using production API: https://maintainhome.ai
```

You can also import the helpers directly:

```ts
import { API_BASE, apiUrl } from "@/lib/api";

// API_BASE === "https://maintainhome.ai" when native, "" in browser
// apiUrl("/api/foo") === "https://maintainhome.ai/api/foo" when native
```

The API server already allows credentialed cross-origin requests
(`cors({ origin: true, credentials: true })`), so cookies sent from the
Capacitor `http://localhost` / `capacitor://localhost` origin will be
honoured by `https://maintainhome.ai`.

## Permissions reference

iOS (`Info.plist`):
- `NSCameraUsageDescription` тАУ camera
- `NSPhotoLibraryUsageDescription` / `NSPhotoLibraryAddUsageDescription` тАУ photos
- `NSDocumentsFolderUsageDescription` тАУ document uploads
- `NSMicrophoneUsageDescription` тАУ future voice notes

Android (`AndroidManifest.xml`):
- `CAMERA`, `RECORD_AUDIO`
- `READ_MEDIA_IMAGES`, `READ_MEDIA_VIDEO` (Android 13+)
- `READ_EXTERNAL_STORAGE` (тЙд 32), `WRITE_EXTERNAL_STORAGE` (тЙд 29)
- `POST_NOTIFICATIONS` (Android 13+) for push
- `VIBRATE`

Push notifications also need:
- **iOS**: APNs key from your Apple Developer account тЖТ uploaded to your
  push provider (Firebase, OneSignal, etc.)
- **Android**: `google-services.json` from Firebase, dropped into
  `android/app/`.
