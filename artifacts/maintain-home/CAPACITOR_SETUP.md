## What you need to do locally

Capacitor needs **Xcode** (Mac only) and **Android Studio** to actually build,
run, and submit. Replit's Linux container can't open these projects, so the
day-to-day native dev loop is on your laptop.

### One-time setup on your machine

1. Pull the repo to your Mac.
2. Install dependencies:
   ```sh
   pnpm install
   ```
3. **iOS only** â€” install Xcode (App Store) and CocoaPods:
   ```sh
   sudo gem install cocoapods
   cd artifacts/maintain-home/ios/App
   pod install
   cd -
   ```
4. **Android only** â€” install [Android Studio](https://developer.android.com/studio)
   and let it install the Android SDK + an emulator.

### Day-to-day workflow

From inside `artifacts/maintain-home`:

```sh
# 1. Build the web bundle and copy it into the native projects
pnpm cap:sync

# 2. Open Xcode (iOS) â€” then click the Play button to run on simulator/device
pnpm cap:open:ios

# 3. Open Android Studio â€” then click the green Play button
pnpm cap:open:android
```

`pnpm cap:sync` runs `vite build` (writes to `dist/public/`) and then
`cap sync`, which copies the web assets and updated plugin native code into
both the `ios/` and `android/` folders.

### Refreshing icons or splash screens

The placeholder icons are generated from `public/images/logo.png`. To swap
them out, drop new source files into `assets/` (`icon-only.png`,
`icon-foreground.png`, `icon-background.png`, `splash.png`,
`splash-dark.png` â€” all square, 1024px+ for icons, 2732px for splash) then
run:

```sh
pnpm cap:assets
pnpm cap:sync
```

(`pnpm cap:assets` requires `sharp`'s native binary; it works on a Mac out
of the box, but on Replit you'd need to use the ImageMagick fallback that
was used for this initial setup.)

## Important â€” API URL in the native app

When the app runs natively, `fetch("/api/...")` resolves to `capacitor://`
or `http://localhost`, **not** `https://maintainhome.ai`. Before submitting
to the stores, make every API call use an absolute production URL via an
env var (e.g. `VITE_API_BASE_URL=https://maintainhome.ai`) or a runtime
detection like:

```ts
import { Capacitor } from "@capacitor/core";

export const API_BASE = Capacitor.isNativePlatform()
  ? "https://maintainhome.ai"
  : "";
```

(That's a small follow-up task â€” flag it before App Store submission.)

## Permissions reference

iOS (`Info.plist`):
- `NSCameraUsageDescription` â€“ camera
- `NSPhotoLibraryUsageDescription` / `NSPhotoLibraryAddUsageDescription` â€“ photos
- `NSDocumentsFolderUsageDescription` â€“ document uploads
- `NSMicrophoneUsageDescription` â€“ future voice notes

Android (`AndroidManifest.xml`):
- `CAMERA`, `RECORD_AUDIO`
- `READ_MEDIA_IMAGES`, `READ_MEDIA_VIDEO` (Android 13+)
- `READ_EXTERNAL_STORAGE` (â‰¤ 32), `WRITE_EXTERNAL_STORAGE` (â‰¤ 29)
- `POST_NOTIFICATIONS` (Android 13+) for push
- `VIBRATE`

Push notifications also need:
- **iOS**: APNs key from your Apple Developer account â†’ uploaded to your
  push provider (Firebase, OneSignal, etc.)
- **Android**: `google-services.json` from Firebase, dropped into
