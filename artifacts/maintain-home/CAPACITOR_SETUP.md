# MaintainHome — Native iOS / Android (Capacitor)

The web app at `artifacts/maintain-home` is wrapped with
[Capacitor 8](https://capacitorjs.com) so it can ship to the **App Store**
and **Google Play** as a native iOS / Android app.

> **TL;DR for the impatient Mac user**
>
> ```sh
> git pull
> pnpm install
> cd artifacts/maintain-home
> cd ios/App && pod install && cd ../..   # iOS first time only
> pnpm cap:sync                            # build web + copy to native
> pnpm cap:open:ios                        # opens Xcode, then click ▶
> pnpm cap:open:android                    # opens Android Studio, click ▶
> ```

---

## What's already in place (done on Replit)

- **Capacitor core + CLI** (`@capacitor/core`, `@capacitor/cli`)
- **Native platforms**: `ios/` and `android/` folders generated by `cap add`
- **Plugins installed**:
  - `@capacitor/camera` — camera + photo library
  - `@capacitor/filesystem` — document upload / local file access
  - `@capacitor/push-notifications` — APNs / FCM push
  - `@capacitor/splash-screen` — branded launch image
  - `@capacitor/app` — app lifecycle events
  - `@capacitor/preferences` — secure key/value storage
  - `@capacitor/status-bar` — colour/style of the status bar
- **Permissions wired up** in `ios/App/App/Info.plist` and
  `android/app/src/main/AndroidManifest.xml`.
- **Placeholder app icon + splash** generated from `public/images/logo.png`.
- **Production API rewrite**: native builds automatically call
  `https://maintainhome.ai`. See "API URL handling" below.
- **Bundle ID**: `com.maintainhome.app`
- **App name**: `MaintainHome`
- **Web bundle dir**: `dist/public`

See `capacitor.config.ts` for the full config.

## 1. One-time setup on your Mac

### A. Install the prerequisites

| Tool | What it's for | How to install |
| --- | --- | --- |
| **Node 20+** | Runs the build | `brew install node` (or use [Volta](https://volta.sh)) |
| **pnpm** | Package manager used by this monorepo | `corepack enable && corepack prepare pnpm@latest --activate` |
| **Xcode 15+** | Builds & submits the iOS app | Install from the **Mac App Store** (≈ 7 GB), then open it once and accept the license |
| **Xcode Command Line Tools** | Required by CocoaPods + Capacitor | `xcode-select --install` |
| **CocoaPods** | iOS dependency manager | `sudo gem install cocoapods` (or `brew install cocoapods`) |
| **Android Studio** | Builds & submits the Android app | Download from <https://developer.android.com/studio> and run the Setup Wizard (installs the Android SDK + an emulator automatically) |
| **Java JDK 17** | Required by recent Android Gradle | Android Studio bundles one — no extra install needed |

After Android Studio's first-run wizard, open **Settings → Languages &
Frameworks → Android SDK** and confirm:

- **SDK Platforms** → Android 14 (API 34) is checked
- **SDK Tools** → "Android SDK Build-Tools 34", "Android SDK
  Command-line Tools (latest)", and "Android Emulator" are checked

### B. Pull the repo and install JS deps

```sh
git pull
pnpm install
```

### C. iOS — install CocoaPods deps

CocoaPods downloads the native Swift/Obj-C dependencies that the
Capacitor plugins rely on. You only need to do this once (and again any
time you add a new plugin):

```sh
cd artifacts/maintain-home/ios/App
pod install
cd -
```

You should see something like `Pod installation complete!`. From now on
`pnpm cap:sync` will keep the Pods up-to-date for you.

### D. Android — let Android Studio sync Gradle

The first time you open `pnpm cap:open:android`, Android Studio will
download Gradle wrappers and SDK packages. Wait for the bottom status
bar to read **"Gradle sync finished"** before clicking ▶.

## 2. Day-to-day workflow

All commands are run from inside `artifacts/maintain-home`.

### Build the web bundle and sync into both native projects

```sh
pnpm cap:sync
```

This runs `vite build` (writes to `dist/public/`, the `webDir` declared
in `capacitor.config.ts`) and then `cap sync`, which copies the web
assets and updated plugin native code into both `ios/` and `android/`.

> **Why `pnpm cap:sync` and not just `npx cap sync`?**
> Our `vite.config.ts` requires `PORT` and `BASE_PATH` env vars. The
> `cap:sync` script presets sensible defaults so you don't have to
> remember them. If you do want to run the underlying commands directly:
>
> ```sh
> PORT=19957 BASE_PATH=/ pnpm run build
> npx cap sync
> ```

### Open the iOS project in Xcode

```sh
pnpm cap:open:ios
```

That opens `ios/App/App.xcworkspace` in Xcode. **Always open the
`.xcworkspace`, never the `.xcodeproj`** — the workspace is what knows
about CocoaPods.

### Open the Android project in Android Studio

```sh
pnpm cap:open:android
```

This opens the `android/` folder. Android Studio will run a Gradle sync;
wait for it to finish.

## 3. Build & run on iOS Simulator

1. Run `pnpm cap:sync` to make sure the latest web build is bundled.
2. Run `pnpm cap:open:ios`.
3. In Xcode's top toolbar, pick a simulator from the device dropdown
   (e.g. **iPhone 15 Pro**). If the dropdown is empty, see the
   troubleshooting section.
4. Click the ▶ Play button (or press **⌘R**).
5. The simulator boots, installs MaintainHome, and launches it.

### Verify the production API rewrite is working

In Safari on your Mac, open **Develop → Simulator → MaintainHome**
(if you don't see "Develop", enable it via **Safari → Settings →
Advanced → Show Develop menu**). In the Web Inspector's **Console** tab
you should see:

```
[Capacitor] Running in native mode — using production API: https://maintainhome.ai
```

In the **Network** tab, every request should go to `maintainhome.ai`,
not `capacitor://` or `localhost`.

### Run on a real iPhone (optional)

1. Plug in your iPhone with a Lightning/USB-C cable. Trust the Mac.
2. In Xcode, select your iPhone from the device dropdown.
3. Open **Signing & Capabilities** for the `App` target. Set
   **Team** to your Apple ID (free Apple ID works for personal builds,
   paid Developer Program required for App Store).
4. Click ▶. The first time, you may need to go to **Settings → General
   → VPN & Device Management** on the iPhone and trust the developer
   certificate.

## 4. Archive and submit to the App Store

1. **Enroll in the [Apple Developer Program](https://developer.apple.com/programs/)**
   ($99/year) and add the team in Xcode's **Settings → Accounts**.
2. Create the app in [App Store Connect](https://appstoreconnect.apple.com)
   with bundle ID `com.maintainhome.app`.
3. In Xcode:
   - Select **Any iOS Device (arm64)** as the destination (top
     toolbar). You cannot archive against a simulator.
   - Bump the version number under the **App** target →
     **General → Identity** if needed (CFBundleShortVersionString and
     Build).
   - **Product → Archive** (this can take 5–15 min).
4. When the Organizer window opens with your archive:
   - Click **Distribute App → App Store Connect → Upload**.
   - Xcode signs and uploads. Wait for the green checkmark.
5. In App Store Connect, fill out the metadata, screenshots, privacy
   labels (camera, photos, microphone, push), and submit for review.

## 5. Build & run on Android Emulator + APK / AAB

### Run on an emulator

1. Run `pnpm cap:sync`.
2. Run `pnpm cap:open:android`.
3. In Android Studio's top toolbar, click the **device dropdown**.
   - If empty: **Tools → Device Manager → Create Device** and pick
     "Pixel 7" with an Android 14 system image.
4. Click the green ▶ button. The emulator boots and installs
   MaintainHome.
5. To inspect the WebView, open Chrome on your Mac, go to
   `chrome://inspect`, and click **Inspect** under the
   `MaintainHome` entry.

### Build a debug APK

```sh
cd artifacts/maintain-home/android
./gradlew assembleDebug
```

Output: `app/build/outputs/apk/debug/app-debug.apk`. You can drag this
onto a connected Android phone or share with testers.

### Build a signed release bundle (AAB) for Google Play

1. In Android Studio: **Build → Generate Signed Bundle / APK…**
2. Choose **Android App Bundle** (Play Store now requires AAB, not APK).
3. Create a new keystore the first time (save it somewhere safe and
   back it up — losing it means you can never update your app):
   - Key store path: `~/keystores/maintainhome-upload.jks`
   - Password: pick a strong one and store it in your password manager
   - Alias: `upload`
4. Pick **release** build variant.
5. Output: `android/app/release/app-release.aab`.
6. Upload to <https://play.google.com/console>.

## 6. Updating icons and splash screens

The placeholder icons are generated from `public/images/logo.png`. To
swap them out:

1. Drop new source files into `artifacts/maintain-home/assets/`:
   - `icon-only.png` — square, **1024×1024 minimum**, transparent or
     solid background
   - `icon-foreground.png` — square logo with **transparent
     background** (used for Android adaptive icons)
   - `icon-background.png` — solid colour or pattern, square, 1024×1024
   - `splash.png` — **2732×2732**, light background
   - `splash-dark.png` — **2732×2732**, dark background
2. Run:
   ```sh
   pnpm cap:assets
   pnpm cap:sync
   ```
3. Rebuild in Xcode / Android Studio.

> `pnpm cap:assets` requires `sharp`'s native binary, which works on a
> Mac out of the box. (On Replit's Linux container, sharp doesn't
> compile cleanly — that's why the initial icons were generated using
> ImageMagick.)

## 8. Troubleshooting

### iOS

**"No simulator devices found" / device dropdown is empty**
Run `xcrun simctl list devices` to see what's installed. In Xcode go to
**Window → Devices and Simulators → Simulators tab → +** and add an
iPhone 15 Pro / iOS 17 simulator.

**`pod install` fails with "Unable to find a specification…"**
Update the CocoaPods spec repo:
```sh
pod repo update
cd artifacts/maintain-home/ios/App && pod install
```

**Build fails with "Signing for 'App' requires a development team"**
In Xcode, select the **App** target → **Signing & Capabilities** →
choose your Apple ID under **Team**. A free Apple ID is enough for
simulator/personal-device builds.

**The app shows a white screen on launch**
You probably forgot to `pnpm cap:sync` after a code change, or the
build wrote to the wrong `webDir`. Confirm `dist/public/index.html`
exists, then re-run `pnpm cap:sync`.

**Permission prompt never appears (camera, photos…)**
iOS only shows the prompt the first time a feature is requested.
Delete the app from the simulator (long-press → ✕) or run
**Device → Erase All Content and Settings** to reset.

### Android

**Gradle sync fails with "SDK location not found"**
Open Android Studio → **Settings → Appearance & Behavior → System
Settings → Android SDK** and note the SDK path. Then create
`android/local.properties` with:
```
sdk.dir=/Users/YOU/Library/Android/sdk
```

**"Emulator: PANIC: Missing emulator engine program for 'x86_64' CPU"**
Open **Device Manager → ⋮ → Edit** and pick a system image whose ABI
matches your Mac (arm64 for Apple Silicon, x86_64 for Intel).

**App installs but immediately crashes**
Run `adb logcat | grep -i capacitor` from
`~/Library/Android/sdk/platform-tools` to see the native error. 99% of
first-run crashes are missing `pnpm cap:sync`.

**Push notifications don't arrive**
You need a Firebase project + `google-services.json` in `android/app/`
and an APNs key in your push provider. See "Push notifications
setup" below.

### Both platforms

**Changes to the web app aren't showing up**
Capacitor bundles the web build into the native app at sync time. Edit
→ `pnpm cap:sync` → rebuild in the IDE. There is no hot reload by
default (you can configure live reload via `capacitor.config.ts` if you
want, but most folks just re-sync).

**API calls return CORS errors**
Confirm the API server is reachable from the device's network and that
`origin: true, credentials: true` is set on the CORS middleware
(it already is in `artifacts/api-server/src/app.ts`). On a physical
device, also make sure you're on Wi-Fi that can reach
`maintainhome.ai`.

## API URL handling in native builds — done

When the app runs natively, `fetch("/api/...")` would resolve to
`capacitor://` (iOS) or `http://localhost` (Android) instead of the real
backend. To avoid touching every component, `src/lib/api.ts` exports a
small `installNativeFetchRewrite()` helper that **monkey-patches
`window.fetch` at app start** and rewrites any relative or
capacitor/localhost URL to `https://maintainhome.ai`. It is a no-op in
the browser/PWA build.

The patch runs once from `src/main.tsx` before React mounts, so every
API call (raw `fetch`, React Query, Axios using fetch adapter, etc.) is
covered automatically. On startup in a native build you'll see this log:

```
[Capacitor] Running in native mode — using production API: https://maintainhome.ai
```

You can also import the helpers directly:

```ts
import { API_BASE, apiUrl } from "@/lib/api";

// API_BASE === "https://maintainhome.ai" when native, "" in browser
// apiUrl("/api/foo") === "https://maintainhome.ai/api/foo" when native
```

The API server already allows credentialed cross-origin requests
(`cors({ origin: true, credentials: true })`), so cookies sent from the
Capacitor `http://localhost` / `capacitor://localhost` origin will be
honoured by `https://maintainhome.ai`.

## Permissions reference

iOS (`Info.plist`):
- `NSCameraUsageDescription` – camera
- `NSPhotoLibraryUsageDescription` / `NSPhotoLibraryAddUsageDescription` – photos
- `NSDocumentsFolderUsageDescription` – document uploads
- `NSMicrophoneUsageDescription` – future voice notes

Android (`AndroidManifest.xml`):
- `CAMERA`, `RECORD_AUDIO`
- `READ_MEDIA_IMAGES`, `READ_MEDIA_VIDEO` (Android 13+)
- `READ_EXTERNAL_STORAGE` (≤ 32), `WRITE_EXTERNAL_STORAGE` (≤ 29)
- `POST_NOTIFICATIONS` (Android 13+) for push
- `VIBRATE`

### Push notifications setup (when you're ready)

- **iOS**: APNs key from your Apple Developer account → uploaded to
  your push provider (Firebase, OneSignal, etc.)
- **Android**: `google-services.json` from Firebase, dropped into
  `android/app/`, plus the Google Services Gradle plugin in
  `android/build.gradle` and `android/app/build.gradle`.
